## 三、问题详细说明

### 🔴 严重问题 1：技术栈与 PDF 要求完全不符

**PDF 明确要求**：
> 采用微信小程序原生框架（WXML/WXSS/JS）开发前端，利用云开发（云函数、云数据库）处理数据交互，确保低开发门槛与高安全性

**实际实现**（见 `server/app.js` 第 1-10 行）：
- 后端使用 `express` + `cors` + `better-sqlite3`
- 数据库使用 `server/db/todo.db` 本地 SQLite 文件
- 完全没有 `wx-server-sdk`、`cloudbase` 相关依赖

**而项目内「系统框架与技术栈说明.txt」却写的是云开发**：
> 云服务：微信云开发（CloudBase）
> 云函数：Node.js 运行时
> 数据库：云数据库（基于 MongoDB 的文档型数据库）

**结论**：**两份文档互相矛盾，实际代码与 PDF 要求不符**。老师或答辩评委翻开启动说明就会立刻发现这个问题。

---

### 🔴 严重问题 2：微信提醒功能未真正实现

**PDF 明确要求**：
> 支持待办到期前 1 小时微信提醒

**实际实现**（见 `server/app.js` 第 40-60 行）：

```
cron.schedule('*/5 * * * *', () => {
  // 扫描到期待办
  todos.forEach(t => {
    // 只是 INSERT reminders 表 + UPDATE todos.reminder_sent = 1
  });
});
```

**问题**：
- 整个系统**没有任何微信订阅消息（subscribeMessage）或模板消息（sendTemplateMessage）调用**
- `server/routes/reminder.js` 只提供了 `/upcoming` 和 `/list` 查询接口
- 前端需要**打开小程序轮询**才能看到提醒，用户实际**收不到微信推送**

**实际效果**：用户退出小程序后，到期的待办不会有任何微信消息到达用户手机。

---

### 🟡 中度问题 3：登录未真正走微信授权

**PDF 要求**：
> 微信授权登录

**实际实现**（见 `server/routes/auth.js` + 启动说明）：
- 接口 `/api/auth/login` 接受任意字符串作为 `openid`
- 启动说明中明确写："也可以手动输入任意字符串作为 openid 登录，系统会自动注册新账号"
- 有 3 个预置 demo 账号（demo_user_001/002/003）

**问题**：
- 没有 `wx.login` → 获取 `code` → 调用 `code2Session` 的完整流程
- 没有真实的 `appid` + `secret` 与微信接口通信
- 这只是**模拟授权**，不是真实的微信授权

---

### 🟢 轻度问题 4：文档内容不一致

- `启动说明.txt`：描述 Node.js + Express + SQLite 的本地部署
- `系统框架与技术栈说明.txt`：描述云开发 + CloudBase + 云数据库

两份文档完全不同的技术栈，答辩时会被质疑。

## 四、建议修复方案

### 方案 A：改论文描述（推荐，工作量小）

改写论文与文档，承认并说明：
1. **技术栈**：改为"本地部署的 Node.js + Express + SQLite 后端"替代云开发
2. **登录**：改为"基于 OpenID 标识的自定义账号系统"替代"微信授权登录"
3. **提醒**：
   - 短期：改写为"提醒事项管理模块（到期扫描 + 提醒记录）"
   - 长期：补充基于小程序订阅消息的真实推送

需要修改的文档：
- `系统框架与技术栈说明.txt`（重写技术栈章节）
- 论文初稿（若已开始撰写）
- 答辩 PPT 中对应描述

### 方案 B：完整按 PDF 要求实现（工作量大）

1. **迁移到云开发**：
   - 后端 5 个路由（auth/todo/category/stats/reminder）拆分为 5 个云函数
   - SQLite 数据全迁移到云数据库（users/todos/categories/reminders 集合）
   - 移除 Express/node-cron，改用云函数定时触发器

2. **接入真实微信授权**：
   - 小程序端调用 `wx.login` 取 `code`
   - 云函数调用 `code2Session` 换取真实 `openid`

3. **接入订阅消息**：
   - 小程序端请求用户授权订阅消息（`wx.requestSubscribeMessage`）
   - 云函数使用 `cloud.openapi.subscribeMessage.send` 推送到期提醒

### 方案 C：折中（推荐重度补强）

保留现有架构，仅补齐最关键的"微信提醒"：
1. 小程序端加入"订阅消息"授权按钮
2. 后端调用微信官方接口直接发送（需要 appid + secret）
3. 文档明确说明为混合架构（本地后端 + 微信订阅消息）
